# Contributing

Thanks for contributing to **cognitive-skills**.

## How to Contribute

1. Fork the repository and create a branch from `main`.
2. Keep changes focused and scoped to a clear improvement.
3. Update relevant docs when behavior or structure changes.
4. Open a pull request with context, rationale, and before/after notes.

## Pull Request Guidelines

1. Use a clear title and description.
2. Link related issues (if any).
3. Include examples/screenshots when changing generated templates or docs output.
4. Ensure no unrelated files are modified.

## Content and Template Changes

1. Preserve role-specific structure and naming conventions.
2. Keep templates deterministic and explicit.
3. Avoid introducing tool/vendor lock-in unless documented.

## Commit Style

Use concise, descriptive commit messages that explain intent.

## Code of Conduct

By participating, you agree to follow our [Code of Conduct](./CODE_OF_CONDUCT.md).
