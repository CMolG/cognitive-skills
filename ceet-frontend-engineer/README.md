# CEET — Frontend Engineer

Interview a frontend engineer, extract their cognitive patterns around UI architecture, state, rendering, accessibility, and taste, and produce a portable AI environment that builds components in their style.

## Who this is for

- Senior / staff frontend engineers who want an AI that matches their taste.
- Design-engineering hybrids who want to codify personal conventions.
- Teams standardizing frontend practice across contributors via AI rules.

## What you'll produce

- `templates/cognitive-clone.md` — how this engineer thinks about UI work.
- `templates/ai-environment.md` — system-prompt-ready, works in Claude / ChatGPT / Cursor / Copilot / Gemini / any tool.

## Framework & stack agnostic

This CEET doesn't assume React, Vue, Svelte, Solid, or anything else. The interview surfaces *framework-independent* cognitive patterns (state philosophy, rendering model, component factoring, a11y reflexes) plus the specific framework the interviewee uses.
